# Week 4 Confidence and Top-k Demo Plan Note

## Context
The future app and pitch should communicate uncertainty, not only a single predicted class.

## Week 4 objective
The objective of this note is to make confidence-aware and top-k outputs part of the evaluation-ready baseline plan.

## Planned outputs
The deployed app and live demo should later show:
- predicted class
- confidence score
- top-k predictions
- explainability-friendly visual output

## Why this matters
These outputs improve transparency, reduce false certainty, and help connect evaluation to deployment and ethics discussions.

## Week 5 handoff direction
In Week 5, these requirements should inform inference formatting and demo-ready result exports.

## Summary
This Week 4 note links operational evaluation work to transparent deployment communication.
