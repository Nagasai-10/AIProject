# Week 4 Baseline Optimisation Placeholder Note

## Context
Week 4 optimisation work should control the first baseline cycle rather than start uncontrolled tuning.

## Week 4 objective
The objective of this note is to document the optimisation placeholders that the first baseline runs will follow.

## Placeholder areas
- optimizer defaults
- scheduler behaviour
- epoch budget
- early stopping logic
- checkpoint policy

## Why this matters
Defined placeholders reduce ad hoc experimentation and make baseline choices easier to explain academically.

## Week 5 handoff direction
In Week 5, these placeholders should become concrete settings attached to the first official runs.

## Summary
This Week 4 note prepares the optimisation stream for controlled baseline implementation.
