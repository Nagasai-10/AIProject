# Week 4 Deployment Guidance Language Note

## Context
As the system approaches baseline implementation, deployment wording must remain responsible and consistent.

## Week 4 objective
The objective of this note is to refine the user-facing language principles that will later shape the app and demo.

## Guidance principles
The deployed system should communicate:
- prediction clearly
- confidence responsibly
- top-k alternatives transparently
- supportive decision guidance
- non-replacement of agronomic expertise

## Why this matters
Poor wording can make a technically good system appear misleading or overconfident. This note reduces that risk before the interface is finalised.

## Week 5 handoff direction
In Week 5, this note should guide interface copy, README wording, and deployment evidence.

## Summary
This Week 4 note keeps deployment messaging aligned with responsible-AI principles and practical user guidance.
