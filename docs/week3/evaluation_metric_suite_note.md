# Week 3 Evaluation Metric Suite Note

## Context
Week 3 evaluation planning should ensure that the project is built around measurable outputs rather than a single late-stage accuracy number.

## Week 3 objective
The objective of this note is to define the core metric suite that later experiments must produce.

## Planned metric suite
The evaluation stream should later include:
- overall accuracy
- macro precision
- macro recall
- macro F1
- per-class performance
- confusion-based interpretation

## Why this matters
A 23-class classification task should not be reduced to one headline number. A richer metric suite is needed to reveal minority-class issues, confusion pairs, and imbalance-related weaknesses.

## Week 4 handoff direction
In Week 4, this metric suite should shape the design of result tables, saved predictions, and reporting outputs.

## Summary
This Week 3 note establishes the evaluation baseline needed for defensible later experiments.
