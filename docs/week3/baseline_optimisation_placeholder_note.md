# Week 3 Baseline Optimisation Placeholder Note

## Context
Week 3 optimisation work should not yet claim tuned results, but it should define the placeholder logic that later baseline runs will follow.

## Week 3 objective
The objective of this note is to prepare controlled placeholders for the first official optimisation path.

## Placeholder areas
The baseline path should later define:
- optimizer
- scheduler
- epoch budget
- early stopping
- checkpoint policy

## Why this matters
Placeholders reduce the risk of ad hoc experimentation. They create a stable starting point for baseline runs and make later optimisation decisions easier to explain.

## Week 4 handoff direction
In Week 4, these placeholders should become explicit configuration choices attached to the first baseline runs.

## Summary
This Week 3 note prepares the optimisation stream for controlled baseline work instead of trial-and-error tuning.
