# Week 3 Result Reporting Checklist

## Context
Evaluation outputs should be planned early so that future experiments automatically generate evidence suitable for the report and pitch.

## Week 3 objective
The objective of this note is to define the reporting artefacts that later experiments should produce.

## Planned outputs
Later experiments should generate:
- classification report
- confusion matrix
- class-wise summary
- training-history plots
- key metric summary table
- example inference output for the demo

## Why this matters
This checklist improves consistency and makes experiments easier to compare across weeks.

## Week 4 handoff direction
In Week 4, these outputs should inform logging templates and result-saving structure.

## Summary
This Week 3 note ensures that later experiments are designed to produce strong reporting artefacts, not only raw predictions.
