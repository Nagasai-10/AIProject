# Week 3 Run Tracking and Persistence Checklist

## Context
A strong training stream needs more than just a notebook that runs. It needs evidence discipline.

## Week 3 objective
The objective of this note is to define the minimum evidence standard for official tracked runs.

## Official run checklist
Each serious run should later produce:
- a configuration snapshot
- a checkpoint
- a metrics summary
- visual outputs where relevant
- clear persistence in Drive or other approved storage
- notes that make the run reviewable later

## Why this matters
This checklist supports:
- reproducibility
- later evaluation
- easier report writing
- stronger deployment preparation

## Week 4 handoff direction
In Week 4, this checklist should become part of the first official baseline-run process.

## Summary
This Week 3 note ensures that later training artefacts are reviewable, persistent, and useful beyond experimentation alone.
