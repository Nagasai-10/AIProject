
# Week 3 Deployment Guidance Language Note

## Context
The deployed system will need clear and responsible user-facing wording. That language should be planned early.

## Week 3 objective
The objective of this note is to define the responsible messaging principles that should shape the app and demo later.

## Planned guidance logic
The deployed system should communicate:
- prediction clearly
- confidence responsibly
- top-k alternatives transparently
- supportive decision guidance
- non-replacement of agronomic expertise

## Why this matters
Poor interface wording can make a technically good system appear misleading or overconfident. Planning guidance language in Week 3 reduces that risk.

## Week 4 handoff direction
In Week 4, this note should help shape README text, interface copy, and deployment evidence.

## Summary
This Week 3 note keeps deployment messaging aligned with responsible-AI principles.
